# 故障排除指南

## 问题：插件按钮点击后没有反应

如果点击插件按钮后没有任何反应，请按照以下步骤逐一检查：

## 🔍 诊断步骤

### 步骤 1: 检查服务器是否运行

**这是最常见的原因！**

#### 检查方法：

1. **查看是否有服务器窗口打开**
   - 应该有一个终端/命令窗口显示"服务器已启动"
   - 窗口标题可能显示"Word Add-in 服务器"

2. **如果没有服务器窗口：**
   - 双击 `启动服务器.command`（macOS）或 `start-server.bat`（Windows）
   - 等待看到"服务器已启动"的提示

3. **如果服务器窗口已关闭：**
   - 重新启动服务器
   - 保持窗口打开

#### 验证服务器是否正常：

在浏览器中访问：
```
https://localhost:3000/taskpane.html
```

- ✅ **如果能看到界面**：服务器正常
- ❌ **如果无法访问**：服务器没有运行或有问题

---

### 步骤 2: 检查浏览器控制台错误

Word Add-in 使用浏览器引擎，可以在浏览器中查看错误信息。

#### macOS 方法：

1. **打开 Safari 浏览器**（Word 在 macOS 上使用 Safari 引擎）

2. **启用开发者菜单**：
   - Safari > 偏好设置 > 高级
   - 勾选"在菜单栏中显示'开发'菜单"

3. **打开 Web 检查器**：
   - 在 Word 中点击插件按钮
   - Safari > 开发 > [你的 Mac 名称] > Microsoft Word
   - 查看控制台标签页中的错误信息

#### 或者使用终端查看日志：

```bash
# 在终端中运行，查看系统日志
log stream --predicate 'process == "Microsoft Word"' --level debug
```

---

### 步骤 3: 检查 manifest.xml 配置

确认 `manifest-local.xml` 中的 URL 是否正确。

#### 检查内容：

1. 打开 `manifest-local.xml` 文件（用文本编辑器）

2. 查找所有 `https://localhost:3000` 的引用

3. 确认：
   - URL 是否正确（应该是 `https://localhost:3000`）
   - 端口号是否匹配（默认是 3000）
   - 如果修改了端口，所有 URL 都要更新

#### 常见错误：

- ❌ URL 写成了 `http://`（应该是 `https://`）
- ❌ 端口号不一致
- ❌ 使用了错误的 manifest 文件（应该用 `manifest-local.xml`）

---

### 步骤 4: 检查 SSL 证书

#### 问题表现：

- 浏览器显示"不安全连接"
- 无法访问 `https://localhost:3000`

#### 解决方法：

1. **首次运行服务器时**：
   - 服务器会自动生成证书
   - 查看插件文件夹中是否有 `localhost.pem` 和 `localhost-key.pem`

2. **在浏览器中信任证书**：
   - 访问 `https://localhost:3000/taskpane.html`
   - 点击"高级"或"Advanced"
   - 选择"继续访问"或"Proceed to localhost"

3. **如果证书有问题**：
   - 删除 `localhost.pem` 和 `localhost-key.pem`
   - 重新启动服务器（会自动重新生成）

---

### 步骤 5: 检查防火墙设置

#### macOS：

1. 系统偏好设置 > 安全性与隐私 > 防火墙
2. 确保防火墙允许本地连接
3. 或者临时关闭防火墙测试

#### Windows：

1. Windows 安全中心 > 防火墙和网络保护
2. 允许应用通过防火墙
3. 确保 Python 或 Node.js 被允许

---

### 步骤 6: 检查端口占用

#### 问题表现：

服务器启动时显示"端口已被占用"

#### 解决方法：

1. **查找占用端口的程序**：

**macOS:**
```bash
lsof -i :3000
```

**Windows:**
```cmd
netstat -ano | findstr :3000
```

2. **关闭占用端口的程序**

3. **或修改端口**：
   - 修改 `scripts/start-server.py`（或离线包内根目录的 `start-server.py`）中的 `PORT = 3000`
   - 同时修改 `manifest-local.xml` 中所有 `localhost:3000` 为新端口

---

### 步骤 7: 重新加载插件

如果以上都正常，尝试重新加载插件：

1. **在 Word 中卸载插件**：
   - 插入 > 加载项 > 我的加载项
   - 找到插件，点击 **...** > **删除**

2. **重新添加插件**：
   - 插入 > 加载项 > 我的加载项
   - 上传我的加载项
   - 选择 `manifest-local.xml`

3. **重启 Word**：
   - 完全退出 Word
   - 重新打开 Word
   - 再次尝试

---

### 步骤 8: 检查 Word 版本

确保 Word 版本支持 Add-in：

- ✅ Microsoft Word for Mac 2016 或更高版本
- ✅ Microsoft 365（推荐）

#### 检查版本：

Word > 关于 Word

---

## 🛠️ 快速诊断清单

按照以下清单逐一检查：

- [ ] 服务器是否正在运行？
- [ ] 浏览器能否访问 `https://localhost:3000/taskpane.html`？
- [ ] manifest-local.xml 中的 URL 是否正确？
- [ ] 使用的是 `manifest-local.xml` 而不是 `manifest.xml`？
- [ ] SSL 证书是否已生成？
- [ ] 防火墙是否阻止了连接？
- [ ] 端口 3000 是否被占用？
- [ ] Word 版本是否支持 Add-in？
- [ ] 是否尝试过重新加载插件？

## 🔧 常见错误及解决方案

### 错误 1: "无法连接到服务器"

**原因：** 服务器没有运行

**解决：** 启动服务器

---

### 错误 2: "证书错误" 或 "不安全连接"

**原因：** 自签名证书未被信任

**解决：** 
1. 在浏览器中访问 `https://localhost:3000`
2. 点击"高级" > "继续访问"
3. 或者在 Keychain Access（macOS）中信任证书

---

### 错误 3: "404 Not Found"

**原因：** URL 路径错误或文件不存在

**解决：**
1. 检查 `manifest-local.xml` 中的 URL
2. 确认所有文件都在正确的位置
3. 确认服务器在正确的目录中启动

---

### 错误 4: "CORS 错误"

**原因：** 跨域请求被阻止

**解决：** 
- 检查服务器代码中的 CORS 设置
- 确保 Python 启动脚本包含 CORS 头设置（`scripts/start-server.py` 或离线包内脚本）

---

### 错误 5: 按钮点击后没有任何反应

**可能原因：**
1. 服务器没有运行（最常见）
2. manifest 文件 URL 错误
3. Office.js 加载失败
4. 网络连接问题

**解决步骤：**
1. 首先检查服务器是否运行
2. 在浏览器中测试 URL 是否可访问
3. 查看浏览器控制台错误
4. 重新加载插件

---

## 💡 预防措施

为了避免问题，建议：

1. **固定服务器位置**：将插件文件夹放在固定位置
2. **创建快捷方式**：方便快速启动服务器
3. **保持服务器运行**：如果经常使用，可以让服务器一直运行
4. **定期更新**：使用最新版本的插件
