# 浏览器控制台诊断命令

在 Safari 控制台中执行以下命令来诊断问题：

## 1. 检查按钮元素

```javascript
// 检查按钮是否存在
document.getElementById('add-rule-btn')
document.getElementById('confirm-btn')
document.getElementById('cancel-btn')
```

**预期结果：** 应该返回按钮元素对象

---

## 2. 检查初始化函数

```javascript
// 检查 initializeAddIn 函数是否存在
typeof initializeAddIn
```

**预期结果：** 应该返回 `"function"`

---

## 3. 手动执行初始化

```javascript
// 手动触发初始化
initializeAddIn()
```

**预期结果：** 
- 应该看到控制台输出："开始初始化插件..."
- 应该看到："✓ 绑定了'添加规则'按钮事件"
- 应该添加第一行规则

---

## 4. 检查事件监听器

```javascript
// 检查按钮是否有事件监听器
const btn = document.getElementById('add-rule-btn');
btn.onclick
```

**预期结果：** 应该返回函数或 null（addEventListener 不会设置 onclick）

---

## 5. 手动触发按钮点击

```javascript
// 方法1: 使用 click() 方法
document.getElementById('add-rule-btn').click()

// 方法2: 创建并触发点击事件
const btn = document.getElementById('add-rule-btn');
const event = new MouseEvent('click', { bubbles: true });
btn.dispatchEvent(event)
```

**预期结果：** 应该添加新的规则行

---

## 6. 检查规则容器

```javascript
// 检查规则容器
const container = document.getElementById('rules-container');
container.children.length
```

**预期结果：** 应该大于 0（至少有一行规则）

---

## 7. 手动添加规则行

```javascript
// 手动调用 addRuleRow 函数
addRuleRow()
```

**预期结果：** 应该添加新的规则行

---

## 8. 检查所有函数

```javascript
// 检查所有关键函数是否存在
typeof addRuleRow
typeof removeRuleRow
typeof collectRules
typeof applyFormatting
```

**预期结果：** 都应该返回 `"function"`

## 10. 完整诊断流程

```javascript
// 完整诊断脚本
console.log('=== 开始诊断 ===');

// 1. 检查元素
console.log('1. 按钮元素:', {
    addRule: document.getElementById('add-rule-btn'),
    confirm: document.getElementById('confirm-btn'),
    cancel: document.getElementById('cancel-btn'),
    container: document.getElementById('rules-container')
});

// 2. 检查函数
console.log('2. 函数存在:', {
    initializeAddIn: typeof initializeAddIn,
    addRuleRow: typeof addRuleRow,
    removeRuleRow: typeof removeRuleRow
});

// 3. 检查初始化状态
console.log('3. 规则行数量:', document.getElementById('rules-container').children.length);

// 4. 手动初始化
console.log('4. 执行初始化...');
if (typeof initializeAddIn === 'function') {
    initializeAddIn();
    console.log('5. 初始化后规则行数量:', document.getElementById('rules-container').children.length);
} else {
    console.error('initializeAddIn 函数不存在！');
}

// 5. 测试按钮
console.log('6. 测试按钮点击...');
const btn = document.getElementById('add-rule-btn');
if (btn) {
    const beforeCount = document.getElementById('rules-container').children.length;
    btn.click();
    setTimeout(() => {
        const afterCount = document.getElementById('rules-container').children.length;
        console.log('7. 点击前:', beforeCount, '点击后:', afterCount);
        if (afterCount > beforeCount) {
            console.log('✓ 按钮功能正常！');
        } else {
            console.error('✗ 按钮点击无效果！');
        }
    }, 100);
} else {
    console.error('按钮不存在！');
}

console.log('=== 诊断完成 ===');
```

## 常见问题

### Q: `initializeAddIn` 返回 `undefined`

**原因：** 函数未定义或代码未执行

**解决：** 
1. 检查控制台是否有 JavaScript 错误
2. 刷新页面
3. 检查 `taskpane.js` 是否正确加载

### Q: 按钮点击无反应

**可能原因：**
1. 事件监听器未绑定
2. `initializeAddIn()` 未执行
3. JavaScript 错误阻止了执行

**解决：**
1. 手动执行 `initializeAddIn()`
2. 然后再次点击按钮
3. 查看控制台错误

### Q: 规则行数量为 0

**原因：** `addRuleRow()` 未执行

**解决：**
1. 手动执行 `addRuleRow()`
2. 检查是否有 JavaScript 错误

---

## 快速修复

如果按钮无反应，尝试：

```javascript
// 1. 手动初始化
initializeAddIn();

// 2. 等待一下
setTimeout(() => {
    // 3. 再次测试按钮
    document.getElementById('add-rule-btn').click();
}, 100);
```

如果这样能工作，说明初始化时机有问题。
