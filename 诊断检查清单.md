# 插件按钮无反应 - 诊断检查清单

## ✅ 快速检查（按顺序）

### 1. 服务器是否运行？ ⭐⭐⭐ 最常见原因

- [ ] 是否有服务器窗口打开？
- [ ] 窗口是否显示"服务器已启动"？
- [ ] 如果没有，双击 `启动服务器.command` 启动

**测试方法：**
在浏览器中访问：`https://localhost:3000/test-server.html`
- ✅ 能看到测试页面 = 服务器正常
- ❌ 无法访问 = 服务器没有运行

---

### 2. 使用的是正确的 manifest 文件？

- [ ] 使用的是 `manifest-local.xml`（不是 `manifest.xml`）
- [ ] manifest 文件中的 URL 是 `https://localhost:3000`

**检查方法：**
1. 打开 `manifest-local.xml`（用文本编辑器）
2. 搜索 `localhost:3000`
3. 确认所有 URL 都是 `https://localhost:3000`

---

### 3. 浏览器控制台是否有错误？

**macOS 查看方法：**
1. 打开 Safari
2. Safari > 偏好设置 > 高级 > 勾选"显示开发菜单"
3. 在 Word 中点击插件按钮
4. Safari > 开发 > [你的Mac] > Microsoft Word
5. 查看控制台标签页

**常见错误：**
- `Failed to load resource` = 文件无法加载
- `CORS error` = 跨域问题
- `Certificate error` = 证书问题

---

### 4. SSL 证书是否正常？

**检查方法：**
1. 在浏览器中访问 `https://localhost:3000`
2. 如果显示"不安全连接"：
   - 点击"高级"或"Advanced"
   - 选择"继续访问"或"Proceed to localhost"

**如果证书文件不存在：**
- 查看插件文件夹中是否有 `localhost.pem` 和 `localhost-key.pem`
- 如果没有，重新启动服务器（会自动生成）

---

### 5. 端口是否被占用？

**问题表现：**
服务器启动时显示"端口已被占用"

**解决方法：**
1. 关闭占用端口的程序
2. 或修改端口号（需要同时修改 manifest 文件）

---

### 6. 是否尝试重新加载插件？

1. Word > 插入 > 加载项 > 我的加载项
2. 找到插件，点击 **...** > **删除**
3. 重新上传 `manifest-local.xml`
4. 重启 Word

---

## 🔍 详细诊断

### 测试服务器连接

1. **打开测试页面**：
   ```
   https://localhost:3000/test-server.html
   ```

2. **点击测试按钮**：
   - "测试服务器连接"
   - "测试文件加载"

3. **查看结果**：
   - 所有测试都通过 = 服务器正常
   - 有测试失败 = 查看具体错误

---

### 收集错误信息

如果问题仍然存在，请收集：

1. **服务器日志**：
   - 启动服务器时终端显示的所有信息
   - 截图或复制文本

2. **浏览器控制台错误**：
   - 按照上面的方法打开控制台
   - 复制所有红色错误信息

3. **系统信息**：
   - macOS 版本
   - Word 版本（Word > 关于 Word）
   - Python 版本（`python3 --version`）

## 💡 常见解决方案

### 问题：按钮点击无反应

**解决方案 1：启动服务器**
- 双击 `启动服务器.command`
- 等待看到"服务器已启动"

**解决方案 2：检查 manifest 文件**
- 确认使用 `manifest-local.xml`
- 确认 URL 是 `https://localhost:3000`

**解决方案 3：重新加载插件**
- 删除旧插件
- 重新上传 manifest-local.xml
- 重启 Word
