# 离线安装指南（用户版）

本指南说明如何在本地使用 Word 插件，无需连接外部服务器。

## 系统要求

- **macOS**: 系统自带 Python 3，无需额外安装
- **Windows**: 需要安装 Python 3（[下载地址](https://www.python.org/downloads/)）
- Microsoft Word for Mac（2016 或更高版本）

## 安装步骤

### 步骤 1: 解压插件包

从管理员获取插件包，解压到任意位置（例如桌面）。

### 步骤 2: 启动本地服务器

#### macOS 用户：

1. 找到解压后的文件夹
2. **双击** `启动服务器.command` 文件
3. 如果提示"无法打开"，右键点击文件，选择 **打开** > **打开**
4. 会打开一个终端窗口，显示服务器运行信息

#### Windows 用户：

1. 找到解压后的文件夹
2. **双击** `start-server.bat` 文件
3. 会打开一个命令窗口，显示服务器运行信息

### 步骤 3: 信任证书（首次使用）

第一次启动服务器时，可能会生成 SSL 证书。在浏览器中访问 `https://localhost:3000` 时：

1. 浏览器会显示安全警告
2. 点击 **高级** 或 **Advanced**
3. 选择 **继续访问** 或 **Proceed to localhost**

**注意：** 这是正常的，因为使用的是自签名证书。

### 步骤 4: 在 Word 中加载插件

1. **打开 Microsoft Word**

2. **加载插件**
   - 点击顶部菜单栏的 **插入**
   - 选择 **加载项** > **我的加载项**
   - 点击 **上传我的加载项** 按钮
   - 浏览并选择解压文件夹中的 `manifest-local.xml` 文件
   - 如果出现安全提示，选择 **信任并加载**

3. **完成安装**
   - 安装完成后，在 Word 的 **开始** 选项卡中会出现 **文本格式化工具** 按钮组

### 步骤 5: 使用插件

1. **保持服务器运行**
   - 服务器窗口必须保持打开状态
   - 不要关闭终端/命令窗口

2. **打开任务窗格**
   - 在 Word 的 **开始** 选项卡中
   - 点击 **显示文本格式化工具** 按钮

3. **使用插件功能**
   - 按照界面提示添加规则并应用格式

## 更新插件

### 如果收到新版本的插件包

1. **停止旧服务器**（如果正在运行）
   - 在服务器窗口中按 `Ctrl+C`

2. **替换文件**
   - 解压新的插件包
   - 替换旧的插件文件夹（或删除旧文件夹，使用新文件夹）

3. **启动服务器**
   - 双击 `启动服务器.command`（macOS）或 `start-server.bat`（Windows）

4. **使用插件**
   - 通常**不需要重新添加加载项**
   - 直接使用即可

**重要提示：**
- ✅ **启动服务器**：每次使用前都需要（这是必须的）
- ❌ **添加加载项**：只需首次安装时做一次，之后不需要（除非 manifest-local.xml 文件有变化）

## 使用说明

### 启动服务器

每次使用插件前，需要先启动本地服务器：

- **macOS**: 双击 `启动服务器.command`
- **Windows**: 双击 `start-server.bat`

### 停止服务器

在服务器窗口中按 `Ctrl+C`（macOS 和 Windows 通用）。

### 关闭服务器窗口

关闭服务器窗口后，插件将无法使用。需要重新启动服务器。

## 常见问题

### Q: 双击启动脚本没有反应？

**macOS:**
- 右键点击文件，选择 **打开** > **打开**
- 或在终端中运行：`chmod +x 启动服务器.command`

**Windows:**
- 检查是否安装了 Python
- 在命令提示符中运行：`python start-server.py`

### Q: 提示"端口已被占用"？

说明端口 3000 已被其他程序使用：

1. 关闭占用端口的程序
2. 或修改 `start-server.py` 中的 `PORT = 3000` 为其他端口（如 3001）
3. 同时修改 `manifest-local.xml` 中所有 `localhost:3000` 为新端口

### Q: Word 提示无法连接？

1. **检查服务器是否运行**
   - 查看服务器窗口是否显示"服务器已启动"

2. **检查防火墙**
   - 确保防火墙允许本地连接

3. **检查证书**
   - 在浏览器中访问 `https://localhost:3000/taskpane.html`
   - 如果能看到界面，说明服务器正常

### Q: 每次都要启动服务器吗？

是的，每次使用插件前都需要启动本地服务器。这是为了提供 HTTPS 服务（Word 插件的要求）。

### Q: 可以设置开机自动启动吗？

可以，但需要一些技术知识：

**macOS:**
- 使用 LaunchAgent 配置自动启动
- 或使用 Automator 创建应用程序

**Windows:**
- 使用任务计划程序
- 或创建快捷方式放到启动文件夹

### Q: 如何卸载插件？

1. 在 Word 中：**插入** > **加载项** > **我的加载项** > 找到插件 > 点击 **...** > **删除**
2. 删除解压的文件夹

## 技术说明

### 为什么需要本地服务器？

Office Add-in 要求使用 HTTPS 协议，不能直接使用本地文件。本地服务器提供 HTTPS 服务，让插件可以在本地运行。

### 安全说明

- 使用的是自签名 SSL 证书，仅用于本地开发
- 不会连接到外部服务器
- 所有数据都在本地处理

## 获取帮助

如果遇到问题：
1. 查看服务器窗口的错误信息
2. 检查系统要求是否满足
3. 联系管理员获取帮助
