# 在浏览器中测试插件的说明

## ⚠️ 重要提示

**直接在浏览器中打开 `https://localhost:3000/taskpane.html` 无法完全正常工作！**

这是因为：
1. **Office.js 需要 Word 环境**：Office.js API 只能在 Office 应用程序（Word、Excel 等）中运行
2. **按钮功能依赖 Office.js**：点击"确认"按钮需要调用 Word API 来操作文档
3. **浏览器中没有 Word 对象**：`Word.run()` 等 API 在浏览器中不存在

## 在浏览器中能看到什么

✅ **可以正常显示：**
- 页面界面和样式
- 输入框和按钮
- 添加/删除规则行

❌ **无法正常工作：**
- 点击"确认"按钮（需要 Word API）
- Office.js 相关功能

## 如何诊断问题

### 步骤 1: 打开浏览器控制台

**Safari:**
1. Safari > 偏好设置 > 高级
2. 勾选"在菜单栏中显示'开发'菜单"
3. 开发 > 显示 Web 检查器
4. 查看控制台标签页

**Chrome/Edge:**
1. 按 `Cmd+Option+I` (macOS) 或 `F12` (Windows)
2. 查看控制台标签页

### 步骤 2: 检查错误信息

在控制台中查找：

#### ✅ 正常情况（在浏览器中）：
```
taskpane.js 开始加载...
错误: Office.js 未加载！
```

这是**正常的**，因为浏览器中没有 Office.js。

#### ❌ 异常情况：
- 红色错误信息
- JavaScript 语法错误
- 文件加载失败

### 步骤 3: 检查按钮元素

在控制台中输入：

```javascript
// 检查按钮是否存在
document.getElementById('confirm-btn')
document.getElementById('add-rule-btn')
document.getElementById('cancel-btn')
```

- 如果返回按钮元素 = 元素存在 ✅
- 如果返回 `null` = 元素不存在 ❌

### 步骤 4: 测试按钮点击

在控制台中输入：

```javascript
// 手动触发按钮点击
document.getElementById('add-rule-btn').click()
```

- 如果添加了新行 = 按钮功能正常 ✅
- 如果没有反应 = 事件未绑定 ❌

### 步骤 5: 检查事件监听器

在控制台中输入：

```javascript
// 检查是否有事件监听器
getEventListeners(document.getElementById('confirm-btn'))
```

（注意：`getEventListeners` 只在 Chrome DevTools 中可用）

## 在 Word 中测试（正确方式）

### 步骤 1: 确保服务器运行

双击 `启动服务器.command`，看到"服务器已启动"

### 步骤 2: 在 Word 中加载插件

1. Word > 插入 > 加载项 > 我的加载项
2. 上传我的加载项
3. 选择 `manifest-local.xml`

### 步骤 3: 打开任务窗格

在 Word 的 **开始** 选项卡中，点击 **显示文本格式化工具** 按钮

### 步骤 4: 查看控制台

1. Safari > 开发 > [你的Mac] > Microsoft Word
2. 查看控制台标签页
3. 应该看到：
   ```
   taskpane.js 开始加载...
   ✓ Office.js 已加载
   ✓ Office.onReady 回调执行
   ✓ 检测到 Word 环境
   ✓ 插件初始化完成！
   ```

## 常见问题

### Q: 在浏览器中按钮没反应？

A: **这是正常的！** 因为：
- Office.js 未加载（浏览器中没有）
- 按钮功能需要 Word API
- 必须在 Word 中使用

### Q: 在 Word 中按钮也没反应？

A: 检查：
1. 服务器是否运行
2. 浏览器控制台中的错误信息
3. 是否看到"插件初始化完成"的日志

### Q: 如何确认代码没问题？

A: 在浏览器中测试：
1. 界面是否正常显示 ✅
2. 能否添加/删除规则行 ✅
3. 控制台是否有 JavaScript 错误 ✅

如果这些都正常，代码就没问题。按钮无反应是因为缺少 Office.js，这是正常的。

## 诊断清单

### 在浏览器中测试（检查代码）：

- [ ] 页面能正常显示
- [ ] 样式正常
- [ ] 可以添加规则行
- [ ] 可以删除规则行
- [ ] 控制台没有 JavaScript 错误
- [ ] 看到"Office.js 未加载"的提示（这是正常的）

### 在 Word 中测试（检查功能）：

- [ ] 服务器正在运行
- [ ] 在 Word 中加载了插件
- [ ] 任务窗格能打开
- [ ] 控制台显示"插件初始化完成"
- [ ] 按钮可以点击
- [ ] 可以应用格式化

## 总结

- **浏览器测试**：用于检查界面和基本 JavaScript 功能
- **Word 测试**：用于检查完整的插件功能
- **按钮在浏览器中无反应是正常的**，因为需要 Office.js

如果：
- ✅ 浏览器中界面正常
- ✅ 控制台没有错误
- ❌ Word 中按钮无反应

那么问题可能是：
1. Office.js 未正确加载
2. 网络问题（Office.js 从 CDN 加载）
3. Word 环境问题

查看 Word 中的浏览器控制台获取详细错误信息。
